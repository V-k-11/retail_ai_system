Retail AI Reorder Prediction System

This project is an end-to-end machine learning engineering system for retail reorder prediction using an Instacart-style dataset. It predicts whether a customer is likely to reorder a product based on order history, product behavior, and cart-level signals.

Business Problem

Retail and grocery platforms need to understand which products a customer is likely to buy again. This supports personalized recommendations, buy-again sections, targeted promotions, inventory planning, and customer retention.

Core question: Will this customer reorder this product?

Output: 1 = likely reordered, 0 = not likely reordered.

Tools, Frameworks, And Why They Are Used

Python

Main programming language for data engineering, machine learning, APIs, and GenAI integrations.

PySpark

Used for scalable data processing: reading large CSV files, converting raw data to Parquet, joining large tables, and aggregating features. PySpark requires Java because Spark executes on the JVM.

Pandas

Used as a lightweight demo engine for small local data. It allows the demo pipeline to run without Java or Spark setup.

Parquet

Used for processed and feature data because it is columnar, compressed, and common in data lake and ML pipelines.

Scikit-Learn

Used to train the baseline Random Forest classifier for tabular reorder prediction.

MLflow

Used for experiment tracking, metric logging, parameter logging, and model lifecycle structure.

FastAPI

Used to serve predictions through HTTP endpoints. It provides /predict, /assistant, /health, /docs, and /demo.

Joblib

Used to save and load the local trained model at data/predictions/retail_model.joblib.

Grok / xAI API

Used by the assistant endpoint when an API key is provided. It explains predictions in business-friendly language.

GitHub Actions

Used for CI/CD validation by running tests on push and pull request.

Render

Used to host the lightweight FastAPI serving demo.

Feature Engineering

The system creates features such as order number, order day of week, order hour, days since prior order, add-to-cart order, user order count, product order count, product reorder rate, aisle ID, and department ID.

These features represent customer behavior, product popularity, timing, and category information.

Model Training

The project currently trains one model type: Random Forest. Training includes train-test split, preprocessing, model fitting, accuracy calculation, ROC-AUC calculation, MLflow logging, and prediction export.

{
  "accuracy": 0.8,
  "roc_auc": 0.75
}

Model Serving

The trained model is served with FastAPI. A request contains customer and product behavior features. The response tells whether the product is likely to be reordered.

{
  "records": [
    {
      "order_number": 10,
      "order_dow": 1,
      "order_hour_of_day": 5,
      "days_since_prior_order": 7,
      "add_to_cart_order": 1,
      "user_order_count": 5,
      "product_order_count": 100,
      "product_reorder_rate": 0.6,
      "aisle_id": 24,
      "department_id": 4
    }
  ]
}
{
  "prediction": 1,
  "label": "likely_reordered",
  "mode": "local_joblib"
}

GenAI Assistant

The GenAI assistant is an explanation layer. The ML model predicts reorder behavior, and the assistant explains the result in business language. This is useful because business users need interpretable recommendations, not only numeric predictions.

Monitoring And Self-Improving Loop

The monitoring module checks if model accuracy falls below a configured threshold. If performance drops, the pipeline can trigger retraining. This is a simple version of production model monitoring.

Local Demo Commands
